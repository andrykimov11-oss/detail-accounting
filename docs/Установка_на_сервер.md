# Установка на сервер предприятия — пошагово

**Для кого:** тот, кто ставит систему в цехе (администратор/технолог).
**Схема:** одна программа на сервере в сети предприятия + телефоны операторов
как браузерные клиенты со сканированием камерой.
**Ниже — под Windows.** На Linux всё то же, отличаются только команды запуска
(`python3` вместо `python`, `/` вместо `\` в путях).

---

## Как это устроено

```
┌──────────────────────────────────────────────┐
│  СЕРВЕР — один компьютер в сети предприятия    │
│  • программа (Python + Flask), порт 5001       │
│  • база prod.db  ← единственное место факта    │
│  • общая папка с выгрузками 1С и Базиса         │
│  IP, например: 192.168.1.50                     │
└───────────────────────┬────────────────────────┘
                        │  Wi-Fi предприятия
          ┌─────────────┼─────────────┐
        📱             📱             📱
     телефон 1      телефон 2      телефон N
     браузер+камера  браузер+камера браузер+камера
```

Программа стоит **на одном компьютере**. База одна — счётчик по заказу общий
для всех телефонов. Телефон просто открывает адрес сервера в браузере и
сканирует QR камерой; на телефон ничего не ставится.

---

## Шаг 1. Выбрать компьютер-сервер

Любой рабочий ПК, который:
- включён во время смен;
- в той же сети (Wi-Fi/LAN), что и телефоны операторов;
- имеет доступ к папкам, куда 1С и Базис кладут выгрузки.

Мощный не нужен — обычного офисного достаточно.

## Шаг 2. Поставить Python

Скачать **Python 3.11** с python.org. При установке отметить галочку
**«Add Python to PATH»**. Проверить в командной строке (`cmd`):

```
python --version
```

Должна показаться версия 3.11.x.

## Шаг 3. Скопировать программу на сервер

Скопировать папку `detail-accounting` (с GitHub или флешкой) на диск сервера,
например в `C:\detail-accounting`.

## Шаг 4. Установить зависимости

В командной строке:

```
cd C:\detail-accounting
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## Шаг 5. Скачать библиотеку сканера (для цеха без интернета)

Камера читает QR библиотекой, которая по умолчанию берётся из интернета.
Если у сервера нет постоянного интернета — один раз, с любого компьютера с
интернетом, скачать файл и положить в
`C:\detail-accounting\src\static\html5-qrcode.min.js`:

```
https://cdn.jsdelivr.net/npm/html5-qrcode@2.3.8/html5-qrcode.min.js
```

(В браузере открыть ссылку → «Сохранить как». Размер ~350 КБ.)
Если у сервера интернет есть — шаг можно пропустить.

## Шаг 6. Создать базу и справочники

```
python main.py --db prod.db init
python main.py --db prod.db rules --areas "C:\путь\Операции по участкам.xlsx"
python main.py --db prod.db import-operators --plan "C:\путь\производственные операции.xlsx" --areas "C:\путь\Операции по участкам.xlsx" --area Кромление
```

- `init` — создаёт базу и участки;
- `rules` — загружает справочник операций (227 операций, 6 участков);
- `import-operators` — заводит операторов кромления из выгрузки 1С.

Пути к заказам и загрузку удобнее делать потом через админку (Шаг 10).

## Шаг 7. Запустить сервер

Задать PIN администратора и запустить:

```
set DA_ADMIN_PIN=ваш_пин
python src\operator_app.py prod.db 5001
```

Сервер напечатает адрес вида `https://<IP>:5001/`. При первом запуске сам
создаст сертификат для HTTPS в папке `certs\` рядом с базой.

**Окно командной строки закрывать нельзя** — пока оно открыто, работает
сервер. Про автозапуск без окна — в конце, раздел «После пилота».

## Шаг 8. Узнать IP сервера и открыть порт

**IP:** в `cmd` команда `ipconfig` → строка «IPv4-адрес», например
`192.168.1.50`. Этот адрес будут открывать телефоны.

**Порт 5001** разрешить в брандмауэре Windows:
Панель управления → Брандмауэр Защитника Windows → Дополнительные параметры →
Правила для входящих подключений → Создать правило → **Порт** → TCP,
конкретный порт **5001** → Разрешить подключение → отметить **Частная** →
имя «Учёт деталей».

## Шаг 9. Подключить телефоны

На телефоне оператора в браузере (Chrome / Safari) открыть:

```
https://192.168.1.50:5001/
```

(подставить реальный IP из шага 8).

Браузер покажет предупреждение **«соединение не защищено»** — это норма для
самоподписанного сертификата в локальной сети. Нажать «Дополнительно» → «Всё
равно перейти». Разрешить доступ к камере. Дальше телефон работает без
вопросов.

Технолог для управления заходит на `https://192.168.1.50:5001/admin`.

## Шаг 10. Настроить пути и загрузить заказы — через админку

1. Открыть `https://192.168.1.50:5001/admin`, ввести PIN.
2. Раздел **«Настройки путей»**: прописать путь к выгрузке 1С
   (производственные операции) и путь к папке с `.xbir` Базиса. Сохранить.
3. Нажать **«Связать и импортировать»** — система свяжет заказы Базис↔1С и
   загрузит связанные. Внизу покажет, сколько заказов и деталей загружено.

Всё. Операторы могут сканировать.

## Работа оператора (телефон)

1. Выбрать себя из списка (без пароля).
2. Выбрать участок (Кромление).
3. Навести камеру на QR бирки — скан сам, со звуком.
4. Смотреть на цветной статус: зелёный «Принято», жёлтый «Повтор» и т.д.
5. Если QR не читается — кнопка «Бирка не читается» → выбрать деталь из списка.

Памятка оператора: `docs/Инструкция_оператора.md`.

## Работа технолога (админка `/admin`)

- **Настройки путей** — где брать выгрузку 1С, Базиса, логи станков.
- **Участки** — справочник, добавление/удаление операций и участков.
- **Операторы** — предзаполнены из 1С; добавление, включение/выключение,
  удаление.
- **Отчёты** — по заказу (план/факт, потери) и по смене (сканы, аномалии,
  по операторам и операциям), выгрузка в Excel для журнала пилота.

## Если что-то не так

| Симптом | Причина / решение |
|---|---|
| Камера не включается | Адрес должен быть `https://`, не `http://`. Разрешить камеру в браузере. |
| «Соединение не защищено» | Норма для самоподписанного сертификата — принять и продолжить. |
| Телефон не открывает адрес | Телефон и сервер в разной сети, либо неверный IP. Проверить `ipconfig`. |
| После перезагрузки телефоны не заходят | Сменился IP сервера — закрепить IP в роутере (по MAC). |
| Камера тормозит | Слабый телефон — работать кнопкой «Бирка не читается» (выбор рукой). |
| Сервер недоступен всем сразу | Проверить, что окно сервера открыто и ПК в сети; порт 5001 разрешён. |
| Забыл PIN админки | Перезапустить сервер с новым `set DA_ADMIN_PIN=...`. |

## Проверка перед пилотом (чек-лист)

- [ ] Python 3.11 поставлен, `python --version` работает.
- [ ] Зависимости установлены (`pip install -r requirements.txt`).
- [ ] Библиотека сканера на месте (если цех без интернета).
- [ ] База создана: `init`, `rules`, операторы заведены.
- [ ] Сервер запускается, печатает адрес `https://...`.
- [ ] IP сервера закреплён, порт 5001 открыт в брандмауэре.
- [ ] Телефон открывает страницу входа, камера включается.
- [ ] В админке заданы пути, «Связать и импортировать» загрузило заказы.
- [ ] Бирки с новым QR напечатаны (ТЗ_бирка_Базис.md).
- [ ] Тестовый скан на кромлении даёт «Принято» и растит счётчик.

## Отладка без камеры (для проверки логики)

Сервер можно поднять по HTTP и слать сканы через API — камера при этом не
работает, но эндпоинты доступны:

```
python src\operator_app.py prod.db 5001 --http
```

---

## После пилота

- **Автозапуск** сервера как службы Windows (через «Планировщик заданий» при
  входе в систему), чтобы не держать окно командной строки открытым.
- **Промышленный сертификат** вместо самоподписанного — уберёт предупреждение
  браузера.
- **Резервное копирование `prod.db`** раз в день — это единственный файл, где
  живёт факт сканирования.
- **Автоматический импорт** заказов по расписанию (планировщик запускает
  `link`+`import`), если ручной запуск через админку окажется неудобным.
