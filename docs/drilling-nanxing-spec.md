# Присадка: нормирование из Nanxing (.SCX)

**Версия:** 1.0
**Дата:** 2026-07-22
**Статус:** реализовано (нормирование по заказу)
**Код:** `src/nanxing_parser.py`

## Зачем

В `.xbir` (Базис-раскрой) отверстий нет — это выгрузка программы оптимизации
раскроя. Участок присадки поэтому не покрывался подетальным учётом
(docs/dataset-validation-report.md). Источник данных о сверлении — отдельная
выгрузка Базиса для станка Nanxing, папка с файлами `.SCX`.

## Формат .SCX

XML: панель с габаритами, список присадочных операций, кромка.

```xml
<Panel ID="12_Цоколь_перед_1135х75_(1_штук)" Length="1135" Width="75" Thickness="16">
  <Machines>
    <Machining Type="1" Face="1" X="151" Y="75" Diameter="5" Depth="36"/>
    <Machining Type="2" Face="5" X="130" Y="37.5" Diameter="5" Depth="11"/>
  </Machines>
</Panel>
```

- `Type`: 1 — отверстие в торец, 2 — отверстие в пласть.
- `Face`: 1-4 торцы, 5-6 пласти.
- Количество деталей — в имени панели: `(2_штук)`.

какого типа и диаметра проходит через участок.

## Разрыв идентификации

**В `.SCX` нет GUID детали.** Панель опознаётся по имени
(`12_Цоколь_перед_1135х75_(1_штук)`), а не по идентификатору из `.xbir`.

Проверка на массиве: пересечение заказов Nanxing с раскроем — 6 из 20,
габариты совпали лишь у 23% панелей. Сопоставление по размерам ненадёжно:
детали одного размера в заказе неразличимы.

**Следствие.** Парсер даёт нормирование присадки **по заказу** (загрузка
участка), а не факт по конкретной детали. Сквозной подетальный учёт на
присадке — статус «эта деталь просверлена» — требует GUID в выгрузке Nanxing.
Это вопрос к настройке Базиса, не к коду.

На пилот кромления это не влияет: присадка вне его периметра.

## Применение

```bash
python main.py --db prod.db drilling --folder "выгрузка Базис/Nanxing"
```

## Что дальше

Для подетального учёта присадки нужен один из вариантов:

1. **GUID в выгрузке Nanxing** — если Базис умеет выгружать идентификатор
   детали в `.SCX`, связка становится однозначной. Предпочтительно.
2. **Связь через позицию в изделии** — номер позиции из имени панели
   (`12_...`) сопоставить с позицией детали. Требует, чтобы нумерация в
   Nanxing и `.xbir` совпадала — проверяется на пилоте присадки.
