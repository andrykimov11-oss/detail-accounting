# Развёртывание для пилота: сервер в цехе, телефоны с камерой

**Дата:** 2026-07-23
**Схема:** один компьютер-сервер в цехе + телефоны операторов как браузерные клиенты

---

## Как это устроено

```
┌─────────────────────────────┐
│  Компьютер-сервер в цехе     │  ← здесь программа и база (prod.db)
│  operator_app.py, порт 5001  │
└──────────────┬──────────────┘
               │  Wi-Fi цеха
      ┌────────┼────────┐
    📱        📱        📱        ← телефоны: открывают адрес сервера
                                    в браузере, сканируют камерой
```

Программа (Python) запускается **на одном компьютере** в сети цеха. База одна
— поэтому счётчик по заказу общий для всех телефонов. Телефон — это просто
браузер, который открывает страницу сервера и сканирует QR камерой. На самом
телефоне ничего устанавливать не нужно.

---

## Требования

**Сервер (любой компьютер в цехе):**

- Python 3.10+;
- пакеты: `pip install -r requirements.txt` (Flask, cryptography, openpyxl);
- в одной Wi-Fi-сети с телефонами;
- знать свой IP в сети цеха (например `192.168.1.50`).

**Телефоны:**

- любой смартфон с камерой и браузером (Chrome / Safari);
- в той же Wi-Fi-сети, что и сервер.

---

## Важно: камере нужен HTTPS

Браузеры дают доступ к камере **только по защищённому соединению** (`https://`).
По обычному `http://` камера не включится. Поэтому сервер запускается с
самоподписанным сертификатом — он создаётся автоматически при первом старте.

Браузер телефона при первом заходе покажет предупреждение «соединение не
защищено» — это нормально для самоподписанного сертификата в локальной сети.
Оператор один раз нажимает «Дополнительно» → «Всё равно перейти», дальше
работает без вопросов.

## Запуск сервера

**1. Подготовить библиотеку сканера (один раз, на компьютере с интернетом):**

```bash
curl -L -o src/static/html5-qrcode.min.js \
  https://cdn.jsdelivr.net/npm/html5-qrcode@2.3.8/html5-qrcode.min.js
```

Если пропустить — интерфейс возьмёт библиотеку с интернета при первой загрузке
(тогда интернет нужен серверу или телефонам в момент открытия страницы).

**2. Подготовить базу пилота:**

```bash
python3 main.py --db prod.db init                     # справочник участков
python3 main.py --db prod.db rules --areas "Операции по участкам.xlsx"
# завести операторов (39 физлиц) — скриптом импорта из выгрузки 1С
python3 main.py --db prod.db link samples/*.xbir --plan "производственные операции.xlsx"
python3 main.py --db prod.db import samples/*.xbir --require-link
```

**3. Запустить сервер:**

```bash
python3 src/operator_app.py prod.db 5001
```

Сервер напечатает адрес вида `https://<IP-этого-ПК>:5001/`. При первом старте
создаст сертификат в папке `certs/` рядом с базой.

**4. На телефоне** открыть в браузере `https://192.168.1.50:5001/` (подставить
реальный IP сервера), принять предупреждение о сертификате, разрешить доступ к
камере.

## Работа оператора

1. Выбрать себя из списка (без пароля).
2. Выбрать участок (Кромление).
3. Навести камеру на QR бирки — скан произойдёт сам, прозвучит сигнал.
4. Смотреть на цветной статус: зелёный «Принято», жёлтый «Повтор» и т.д.
5. Если QR не читается — кнопка «Бирка не читается» → выбрать деталь из списка.

Подробнее — `docs/Инструкция_оператора.md`.

---

## Если что-то не так

| Симптом | Причина / решение |
|---|---|
| Камера не включается | Адрес должен быть `https://`, а не `http://`. Разрешить камеру в браузере. |
| «Соединение не защищено» | Норма для самоподписанного сертификата — принять и продолжить. |
| Телефон не открывает адрес | Телефон и сервер в разной Wi-Fi-сети, либо неверный IP. |
| Камера тормозит | Слабый телефон — использовать кнопку «Бирка не читается» (выбор рукой). |
| Сервер недоступен всем сразу | Проверить, что компьютер-сервер включён и в сети. |

---

## Отладка без камеры

Для проверки логики без телефонов сервер можно поднять по HTTP и слать сканы
через API:

```bash
python3 src/operator_app.py prod.db 5001 --http
```

Камера в этом режиме не работает (нужен https), но эндпоинты `/api/scan` и
прочие доступны для тестов.

## Что дальше, после пилота

- **Промышленный сертификат** вместо самоподписанного (чтобы не принимать
  предупреждение) — если решите масштабировать.
- **Автозапуск сервера** как службы, чтобы не запускать вручную каждое утро.
- **Резервное копирование `prod.db`** — это единственное место, где живёт факт.
