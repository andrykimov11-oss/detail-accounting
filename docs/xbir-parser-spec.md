# Спецификация парсера `.xbir` → модель `Detail`

**Версия:** 0.1
**Дата:** 2026-07-08
**Статус:** утверждён для реализации

## Назначение

Парсер извлекает подетальные данные из XML-файла `.xbir` (выгрузка программы
«Базис-раскрой») в нормализованную модель детали, пригодную для подетального
учёта производства корпусной мебели из ЛДСП.

## Входной формат

Файл `.xbir` — XML, кодировка UTF-8 (с BOM).

Структура:
```
<Document>
  <Version>1.0</Version>
  <Cols>
    <Col Name="…" Type="Number|String" Index="N"/>   <!-- 128 колонок -->
    …
  </Cols>
  <Rows>
    <Row>зн1⇥зн2⇥…⇥зн128</Row>                        <!-- таб-разделённые значения -->
    …
  </Rows>
</Document>
```

Порядок значений в `<Row>` соответствует `Index` колонок (1..N).
Набор колонок может меняться между версиями Базиса — парсер обязан работать по
имени, а не по позиции.

## Выходная модель `Detail`

| Поле модели        | Колонка .xbir                          | Тип      | Преобразование / правило                          |
|--------------------|----------------------------------------|----------|---------------------------------------------------|
| `detail_uid`       | `ID детали`                            | str      | GUID, непустой; первичный ключ детали             |
| `order_num`        | `Номер заказа`                         | int      | regex: первое целое (`"6564 Спецторг ООО"`→`6564`)|
| `order_raw`        | `Номер заказа`                         | str      | исходная строка (для отладки/аудита)              |
| `material_name`    | `Материал`                             | str      | как есть                                          |
| `material_article` | `Артикул материала`                    | str      | как есть                                          |
| `thickness`        | `Толщина`                              | int      | мм, > 0                                           |
| `length`           | `Длина детали`                         | int      | мм, > 0                                           |
| `width`            | `Ширина детали`                        | int      | мм, > 0                                           |
| `qty`              | `Кол-во`                               | int      | > 0; сколько одинаковых деталей в заказе          |
| `perimeter`        | `Периметр`                             | float    | м                                                 |
| `area`             | `Площадь`                              | float    | м²                                                |
| `volume`           | `Объем`                                | float    | м³                                                |
| `edge_l1/l2/w1/w2` | `Кромка L1/L2/W1/W2 толщ.`             | float    | 0 если пусто/нет кромки                           |
| `edge_total_len`   | расчётное                              | float    | сумма длин сторон с ненулевой кромкой (м)         |
| `drill_through`    | `Кол. скв. отв. в пласть`              | int      |                                                   |
| `drill_blind`      | `Кол. гл. отв. в пласть`               | int      |                                                   |
| `drill_end`        | `Кол. отв. в торец`                    | int      |                                                   |
| `grooves`          | `Кол. прямол. пазов`+`Кол. кривол. пазов` | int  | сумма                                             |
| `plate_no`         | `Номер плиты`                          | int      |                                                   |
| `map_no`           | `Номер карты`                          | int      |                                                   |
| `pos_no`           | `Номер позиции`                        | str      | позиция в изделии                                 |
| `source_file`      | —                                      | str      | путь к .xbir (аудит)                              |

## Правила валидации

1. `detail_uid` непустой и является GUID-подобной строкой.
2. `order_num` извлекается регуляркой `\d+` из `order_raw`; если не найдено — ошибка.
3. `qty > 0`, `length > 0`, `width > 0`, `thickness > 0`.
4. Количество распарсенных строк = количеству `<Row>` (контроль полноты).
5. Все имена колонок из спецификации присутствуют в файле; если нет — warning
   (формат мог измениться), но парсинг продолжается с пустыми значениями.
6. Неизвестные колонки (не из спецификации) игнорируются, но фиксируются в отчёте.

## Что НЕ извлекается (избыточно для учёта)

- Эскиз `.wmf`, пути к шаблонам бирки `.brx`.
- Поля `%…` (служебные Базиса).
- Внутренние координаты панели `X панели`, `Y панели`.
- Облицовки лицевой/нелицевой пласти (фронт-задача, не влияет на операции цеха).

## Выходные артефакты

1. `out/details.csv` — плоская таблица всех деталей (одна строка = одна запись в .xbir).
2. `out/validation_report.json` — отчёт: кол-во строк, ошибок, предупреждений,
   список неизвестных колонок, список строк с ошибками.

## Границы ответственности (что парсер НЕ делает)

- НЕ связывает с 1С (это агент-интегратор).
- НЕ собирает факт выполнения операции (это агент сбора факта).
- НЕ нормализует GUID в штрихкод (это агент бирки).
- НЕ интерпретирует `qty > 1` как физические экземпляры (хранит как поле; развёртка
  в физические экземпляры — отдельная операция модели учёта).
